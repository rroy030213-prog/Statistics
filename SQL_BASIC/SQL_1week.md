## 1-1. BigQuery 기본지식

~~~
✅ 학습 목표 :
* 데이터 관련 기초 지식(OLTP, SQL, Row, Column, 저장 형태 등)을 설명할 수 있다. 
* BigQuery 관련 기초 지식에 대해서 파악할 수 있다. 
~~~

OLTP(Online Transaction Processing): 거래를 하기 위해 사용되는 데이터베이스
데이터가 무결하고 데이터의 추가, 변경이 많이 발생
SQL을 사용해 데이터를 추출할 수 있지만 쿼리 속도가 느릴 수 있다.

SQL: 데이터베이스에서 데이터를 가지고 올 때 사용하는 언어

Row: 새로운 row는 가로로 한 줄을 의미, 하나의 row가 고유한 데이터


Column: 열, 각 데이터의 특정 속성 값

BigQuery: Google Cloud의 OLAP(분석을 위한 기능 제공) + Data Warehouse(데이터를 한 곳에 모아서 저장)
장점: SQL을 사용해 쉽게 데이터 추출 가능
 - OLAP도구이므로 속도가 빠름
 - Firebase, Google Analytics4의 데이터를 쉽게 추출할 수 있음
 - 별도의 서버를 띄울 필요 없이 구글에서 인프라를 관리함

## 2-2. 저장된 데이터 활용하기

~~~
✅ 학습 목표 :
* 데이터가 저장되는 형태를 알고 저장된 데이터를 활용할 수 있다. 
~~~

ERD: 데이터베이스 구조를 한눈에 알아보기 위해 사용
형태: 여러개의 테이블이 있고 테이블마다 칼럼이 있는데 칼럼이 서로 다른 테이블에 중복으로 들어가있을 수 있음.

ERD가 없다면 모든 데이터베이스를 직접 보면서 탐색하기
 - 어떤 테이블이 존재하는지
 - 어떤 컬럼이 존재하는지
 - 다른 테이블과 연결할 때 어떤 컬럼을 사용하는지
 - 컬럼의 값들은 어떤 의미를 가지는지 

 회사에 존재할 수 있는 데이터
1) 서비스에 사용될 데이터베이스 ex) 유저 테이블, 배송 테이블, 물건 테이블
2) 앱/웹 로그 데이터
3) 공공 데이터, 서드파티 데이터 ex) 날씨, 페이스북 광고 데이터

## 문제 2

> **🧚Q. 이번 강의를 통해 SQL이 왜 필요하다고 느끼는지, SQL을 통해 본인이 어떤 것을 해내고 싶은지를 자유롭게 작성해보세요.**

~~~
SQL은 데이터를 분석하기 전에 데이터베이스에서 데이터를 가져오고 가져온 데이터를 필터링, 추출, 변환, 요약 등을 통해 더 분석하기 쉽게 만드는 역할을 합니다. SQL을 통해 데이터를 더 보기 쉽게 만들수 있고 문제해결에 있어서 왜 이런식으로 상호작용하는지 분석해보며 데이터 결과를 검증할 수 있습니다. SQL을 통해 데이터를 분석하기 전에 칼럼들을 분석하기 쉽게 정리하고, 각 칼럼들을 보며 어떤 연관성이 있는지 보며 가설을 세우고 검증하는 과정을 경험해보고싶습니다.
~~~
